Detailed Description

template<typename K, typename V, typename Hash = VaHash<K>>
class VaDict< K, V, Hash >

dictionary (map) that maintains insertion order. This class implements a dictionary (key-value store) using a hash table with separate chaining.

Template Parameters
KThe key type (must support equality comparison and hashing).
VThe value type.
HashThe hash function type (defaults to VaHash<K>).
Note
it preserves the order in which elements were inserted.

Member Typedef Documentation

◆ RawView

template<typename K , typename V , typename Hash = VaHash<K>>
using VaDict< K, V, Hash >::RawView = VaDictRawView<K, V, Hash>

Alias for a raw view of the dictionary's internal state. This struct provides a layout-compatible snapshot of the hash map's internals, usable for both inspection and unsafe mutation.

Note
The RawView type is layout-compatible with VaDict and provides direct access to the internal fields such as buckets, size, and capacity. This is useful for advanced use cases like serialization or debugging.

◆ getRawView()

template<typename K , typename V , typename Hash = VaHash<K>>
const RawView * VaDict< K, V, Hash >::getRawView ( ) const
inline

Returns a safe, read-only view of the dictionary internals. This method returns a pointer to the current dictionary object, reinterpreted as a const-qualified raw view. It allows inspection of the internal state without enabling mutation.

Returns
A const pointer to a VaDictRawView<K, V, Hash>.
Note
The returned view is valid only as long as the owning dictionary (VaDict) exists. Accessing the view after the dictionary is destroyed results in undefined behavior.
This cast is valid because the layout of VaDict begins with fields that are identical to those in VaDictRawView.
Warning
This method does not perform any synchronization. If the dictionary is modified concurrently, the view may become inconsistent.
See also
getUnsafeAccess() for mutable access.

RawView * VaDict< K, V, Hash >::getUnsafeAccess ( )
inline

Grants unsafe mutable access to the dictionary internals. This method returns a raw pointer to this object reinterpreted as a VaDictRawView<K, V, Hash>. All members (buckets, size, hash) can be modified.

Returns
A mutable pointer to a VaDictRawView<K, V, Hash>.
Note
The returned view is valid only as long as the owning dictionary (VaDict) exists. Accessing the view after the dictionary is destroyed results in undefined behavior.
Warning
This is an unsafe interface. The caller is responsible for maintaining all invariants of the dictionary. Incorrect use may result in memory corruption, undefined behavior, or crashes.
This method does not perform any synchronization. Concurrent modifications to the dictionary while using the view may lead to race conditions or corruption.
See also
getRawView() for a read-only alternative.
